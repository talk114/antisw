Phần 01 · Khái niệm

Anti Gravity Switcher là gì?

Một desktop app gateway chạy cục bộ trên máy bạn. Nó kết nối các tài khoản AI doanh nghiệp với Claude Code CLI và Antigravity IDE — bạn không cần tự cấu hình token, không cần proxy ngoài.

🔐

VNPAY SSO

Đăng nhập một lần qua genai.vnpay.vn. JWT được gắn vào ~/.claude/settings.json tự động.

⚡

Opus / Sonnet 4.6

Truy cập model cao cấp qua hạ tầng VNPAY thay vì gói cá nhân — không lo hết quota giữa giờ.

🖥️

Cục bộ & an toàn

Mọi cấu hình nằm trên máy bạn. Không gửi prompt qua bên thứ ba ngoài VNPAY GenAI gateway.

💡

Tóm gọn: Mở app → đăng nhập VNPAY SSO 1 lần → Claude Code CLI / Antigravity IDE chạy ngon, không cần biết về token, JWT, hosts file.

Toàn bộ thao tác hằng ngày gói trong một thanh công cụ duy nhất. Phía dưới là danh sách tài khoản dạng list hoặc grid.

Bốn nút màu nổi bật ở phải — CLI Claude / CLI VNPAY / On Antigravity / SYNC AG — là 4 tác vụ duy nhất bạn cần thuộc lòng. Slide tiếp theo sẽ đi từng nút.

Phần 03 · 4 nút action · 1/4

 Đăng ký slot Claude MAX

Khi bấm, app gửi machine_name + IP + HMAC-SHA256 lên genai.vnpay.vn/auth-cli để xác thực thiết bị của bạn vào pool Claude MAX. Bạn chỉ cần đăng nhập Google trên trình duyệt.

Bấm nút CLI Claude

App tự lấy hostname + IP nội bộ, ký HMAC và mở browser sang trang xác thực VNPAY.

Đăng nhập Google trên browser

Hoàn tất Google OAuth. VNPAY ghi nhận máy của bạn vào slot.

Quay lại app, dùng Claude Code CLI

Gõ claude trong terminal — slot đã hoạt động, không cần token thủ công.

⚠️

Lỗi 400 khi xác thực thường có nghĩa bạn chưa được thêm vào slot. Báo nhóm GenAI để cấp quyền.

Phần 03 · 4 nút action · 2/4

 Cài JWT VNPAY vào Claude Code

Nút này ghi JWT của bạn vào ~/.claude/settings.json. Sau đó Claude Code CLI sẽ đi qua gateway VNPAY thay vì Anthropic gốc — dùng Opus / Sonnet 4.6 free trong nội bộ.

▶

Khi chưa cài (nút xanh)

Bấm CLI VNPAY → SSO → JWT được ghi tự động. Toast báo "CLI VNPAY đã cấu hình vào settings.json".

⏏

Khi đã cài (nút đỏ "GỠ CLI VNPAY")

Bấm sẽ xoá JWT khỏi settings.json, trả Claude Code về cấu hình gốc.

💡

Khi nào dùng: Nếu bạn làm việc qua Claude Code CLI / Claude Desktop. Không cần dùng nếu chỉ làm việc trên IDE Antigravity (xem nút kế tiếp).

Phần 03 · 4 nút action · 3/4

 Bật chế độ VNPAY cho Antigravity IDE

Antigravity IDE không hỗ trợ đổi endpoint trực tiếp, nên app dùng hosts redirect + MITM nội bộ để chuyển traffic qua VNPAY. Cần mật khẩu sudo để chỉnh hosts file.

Bấm On Antigravity

Nếu chưa SSO, app mở browser → bạn xác thực VNPAY → quay lại app.

Nhập mật khẩu sudo

App ghi entries vào /etc/hosts và khởi động MITM proxy cục bộ. Toast báo PID khi thành công.

Mở Antigravity IDE bình thường

Toàn bộ request từ IDE đi qua VNPAY mà bạn không phải đổi cấu hình IDE.

Tắt khi xong việc → Undo AG

Nút chuyển đỏ. Bấm và nhập sudo lần nữa để khôi phục hosts file gốc.

Phần 03 · 4 nút action · 4/4

 Đồng bộ tài khoản từ VNPAY

Bấm để kéo danh sách tài khoản Gemini / Claude bạn đã được phân bổ trên VNPAY về app cục bộ — không cần copy refresh-token thủ công.

📥

Khi nào bấm?

Lần đầu mở app, hoặc sau khi admin VNPAY cấp thêm tài khoản cho bạn.

🔁

Có gì xảy ra?

App mở SSO → VNPAY trả về danh sách qua callback nội bộ → tài khoản hiển thị ngay trên list.

⚠️

Nếu thấy thông báo unknown or expired state — chỉ đơn giản là phiên SSO hết hạn. Bấm lại nút SYNC AG để bắt đầu mới.

Phần 03 · Tóm tắt

Khi nào bấm nút nào?

🖥 CLI Claude

Một lần khi nhận máy mới

Đăng ký thiết bị vào slot Claude MAX. Sau khi xong là quên đi — chỉ làm lại nếu đổi máy.

⚙ CLI VNPAY

Bạn dùng Claude Code CLI

Bấm 1 lần để cài JWT. Bấm GỠ khi muốn quay về Claude gốc (hiếm khi cần).

⚡ On Antigravity

Bạn dùng Antigravity IDE

Bật trước khi bắt đầu work session. Tắt khi xong để hosts file sạch.

🔄 SYNC AG

Khi danh sách tài khoản trống / cũ

Kéo về danh sách mới nhất từ VNPAY. An toàn để bấm bất cứ lúc nào.

Phần 04 · Quản lý tài khoản

Lọc, tìm kiếm và đổi view

Danh sách tài khoản có 2 chế độ hiển thị và 4 bộ lọc tier. Tất cả đều lưu lại lựa chọn của bạn cho lần mở app sau.

☰

List vs Grid

🏷️

Tìm theo email

💡

Khi đang chọn nhiều tài khoản, bộ lọc và search vẫn hoạt động — phần checkbox của bạn được giữ nguyên cho từng id.

Phần 04 · Thao tác hàng loạt

Chọn nhiều tài khoản → 3 nút action xuất hiện

🗑

Xoá hàng loạt

Có dialog xác nhận. Tài khoản bị xoá khỏi store cục bộ — cần SYNC AG nếu muốn lấy lại.

⏸

Tạm tắt proxy

Đưa tài khoản ra khỏi rotation tạm thời (vd. khi nghi ngờ token lỗi). Có thể bật lại bằng nút xanh.

▶

Bật lại proxy

Đưa hàng loạt tài khoản đã tắt trở về rotation. Toast báo số lượng đổi trạng thái.

⚠️

Quy tắc đội nhóm: Không xoá tài khoản dùng chung trừ khi đã xác nhận với admin GenAI. Khi nghi ngờ, dùng tắt proxy để cô lập thay vì xoá.

Phần 04 · Refresh & Import

Quota, hiển thị, import JSON

↻

Refresh quota

Không chọn ai → refresh tất cả. Có chọn → chỉ refresh những id đã chọn. Toast báo số success / failed.

👁

Show all quotas

Toggle để hiện hạn mức của tất cả model trên mỗi account, không chỉ model đang dùng. Hữu ích để chẩn đoán nhanh.

⬆

Import JSON

Nhập batch tài khoản từ file JSON hợp lệ (refresh_token bắt đầu bằng 1//). Báo cáo số lượng partial / fail.

🎯

Best practice: Refresh quota mỗi sáng trước khi vào việc. Nếu nhiều account báo "Vô hiệu" cùng lúc — kiểm tra mạng / SSO trước khi nghĩ tới việc xoá.

Phần 05 · Kỹ thuật dùng AI · 1/4

Chọn đúng model cho đúng việc

Lãng phí Opus cho công việc nhỏ là một trong các nguyên nhân pool quota cạn nhanh nhất. Hướng dẫn theo task:

🧠

Opus 4.6 / 4.7

Reasoning sâu — refactor đa file, debug bug khó, design system, viết spec phức tạp. Đắt nhất, dùng có chủ đích.

⚖

Sonnet 4.6

Mặc định cho coding hằng ngày — đọc code, viết test, sửa bug vừa, giải thích. Cân bằng tốc độ & chất lượng.

⚡

Haiku / Flash

Tác vụ nhanh — tóm tắt, đặt tên biến, format text, tự động hoá nền. Đừng dùng cho task cần tư duy.

💡

Antigravity IDE và Claude Code đã có cơ chế "hạ cấp thầm lặng": tác vụ nền (đặt tên hội thoại, autocomplete) tự rớt về Flash. Bạn chỉ cần lo task chính của mình.

Phần 05 · Kỹ thuật dùng AI · 2/4

Viết prompt — 4 thói quen đáng tiền

🎯

1. Nói rõ đầu ra

"Trả về JSON với field X, Y, Z" tốt hơn "tóm tắt giúp tôi". Định dạng đầu ra rõ → ít round-trip → ít token.

📎

2. Cung cấp ngữ cảnh tối thiểu cần

Dán đoạn code liên quan, không dán cả file 5000 dòng. Nhiều rác → AI tập trung sai chỗ.

🪜

3. Chia nhỏ task

"Đọc → đề xuất → tôi xem → mới sửa". Đừng đưa "làm hết" cho prompt một dòng. Bạn mất kiểm soát, AI mất context.

🧪

4. Yêu cầu nó verify

"Sau khi sửa, chạy test/lint và báo lại". Cho AI làm việc của tester — bắt nó tự bắt lỗi trước khi giao bạn.

Phần 05 · Kỹ thuật dùng AI · 3/4

Nên / Không nên

NÊN

  • Đọc lại diff AI gen ra trước khi commit — nó có thể "hợp lý mà sai".
  • Hỏi AI giải thích đoạn code lạ thay vì copy-paste mò.
  • Dùng AI để viết unit test cho code có sẵn — nhanh và an toàn.
  • Tận dụng cache prompt (gateway VNPAY hỗ trợ): giữ system prompt ổn định, biến phần thay đổi ra cuối.
  • Báo nhóm GenAI khi gặp lỗi lặp lại — có thể là vấn đề pool, không phải bạn.

KHÔNG NÊN

  • Dán thông tin bí mật khách hàng, secret/API key, dữ liệu sản xuất vào prompt.
  • Tin AI 100% với câu trả lời về số liệu cụ thể, định danh, version — luôn check lại nguồn.
  • Mở 5 cửa sổ Antigravity cùng lúc để "chạy nhanh hơn" — chỉ cạn quota chung.
  • Spam refresh quota mỗi 5 giây — gateway có rate limit, sẽ phản tác dụng.
  • Xoá tài khoản dùng chung khi thấy lỗi 403/Vô hiệu — báo nhóm trước.
Phần 05 · Kỹ thuật dùng AI · 4/4

Vài pattern thực chiến

🔍

Pattern "Plan → Approve → Apply"

Yêu cầu AI chỉ nêu kế hoạch trước, bạn duyệt, rồi mới cho phép sửa code. Tránh AI đi quá xa.

🧩

Pattern "1 file 1 prompt"

Khi refactor, cô lập 1 file mỗi lần. Diff nhỏ → review nhanh → ít regression.

🧪

Pattern "Red → Green"

Viết test fail trước, đưa AI sửa code cho đến khi xanh. Quy chuẩn rõ, AI khó "ăn gian".

📝

Pattern "Save as recipe"

Prompt nào dùng nhiều — lưu thành snippet (CLAUDE.md, AGENTS.md hoặc Notion team). Đừng gõ lại mỗi sáng.

📚

Đầy đủ doc & FAQ tại https://gravityland.vnoffice.io.vn/support.html — bookmark sẵn để dùng khi gặp lỗi.

Phần 06 · Xử lý sự cố nhanh

Lỗi thường gặp & cách xử

🛑 Tài khoản báo 403

Tài khoản Google cá nhân không có quyền Gemini Code Assist. App tự bỏ khỏi rotation. Báo admin GenAI nếu là account công ty.

🔁 invalid_grant trong Claude Code

Token Google OAuth hết hạn. Vào trang Tài khoản → tìm tài khoản "Vô hiệu" → xác thực lại. Bật Smart Warmup ở Settings để tránh tái diễn.

❌ unknown or expired state

Phiên SSO hết hạn giữa chừng. Bấm lại nút (CLI VNPAY / On Antigravity / SYNC AG) từ đầu là xong.

🔒 Antigravity vẫn vào endpoint cũ

Kiểm tra nút có đang ở trạng thái đỏ "Undo AG" không. Nếu chưa — bấm "On Antigravity" và nhập sudo. Nếu sai pass, hosts file không được sửa.
